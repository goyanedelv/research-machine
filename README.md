# Academic Research Machine 🎓🤖

> **Note:** AI may make mistakes and occasionally hallucinate. The purpose of this system is to profile and assist with articulated literature reviews. It should **not** replace academic rigor, peer review, or your own critical thinking.

An autonomous, multi-agent AI pipeline designed iteratively to research, verify, and write rigorous academic papers locally. Built specifically using Antigravity agents/skills workflows.

## Features
- **Web-First Context Gathering**: Automatically seeks out academic materials and reads deep web context.
- **Tavily MCP Integration**: Relies heavily on the Tavily Model Context Protocol (MCP) server for high-quality, source-grounded web research and academic discovery.
- **Hallucination Prevention**: Explicit Verification phase tests metadata (DOIs, Links) to ensure real references.
- **Professional Formatting**: Emits papers cleanly in structure (`Abstract` to `References`).

## Directory Structure

```
research-machine/
├── research_notes/      # Notes, scraped abstracts, and URLs aggregated by the Web Research Agent.
├── references/          # Verified and formatted APA bibliographies produced by the Verification Agent.
├── output_document/     # Synthesized academic Markdown papers generated by the Writing Agent.
├── .agent/
│   ├── skills/          # Contains the instructions for the various sub-agent "personas":
│   │   ├── web_research_agent/SKILL.md
│   │   ├── reference_verification_agent/SKILL.md
│   │   ├── writing_agent/SKILL.md
│   │   └── orchestrator_agent/SKILL.md
│   └── workflows/       
│       └── generate_paper.md   # The master orchestrator flow.
└── README.md            # This documentation.
```

## How to Work the System

To start the academic research pipeline, you need to trigger the **Orchestrator Agent**, which is the central coordinator of the machine. The orchestrator is driven by the `generate_paper` workflow.

You can trigger the entire pipeline in the Antigravity chat interface in two primary ways:

**Method 1: Using the Slash Command (Recommended)**
Since workflows are automatically registered as slash commands, type the following directly into the chat:
> `/generate_paper write an academic paper about [Your Topic Here]`

**Example:**
> `/generate_paper write an academic paper about impact of artificial intelligence on the labor market`

*(Sample files demonstrating the output of the example above can be found in `output_document/sample_output.md`, `references/sample_references.md`, and `research_notes/sample_notes.md`).*

**Method 2: Natural Language Invocation**
You can also explicitly instruct Antigravity to use the orchestrator:
> "Use the `orchestrator_agent` skill to run the `generate_paper` workflow and write a paper about [Your Topic Here]"

Once triggered, you do not need to manually invoke any other agents. The Orchestrator will autonomously instruct Antigravity to step through the `web_research_agent`, `reference_verification_agent`, and `writing_agent` phases sequentially, ultimately producing a completely sourced and formatted markdown paper securely in the `output_document/` directory.

## Advanced Overrides
- You can provide custom constraints (e.g., "Use at least 10 sources" or "Write in IEEE format") at the start, and the Orchestrator will cascade them through the pipeline.
- Antigravity's `browser_subagent` may be automatically utilized if the topic requires downloading or analyzing visual / authenticated portals if accessible.
